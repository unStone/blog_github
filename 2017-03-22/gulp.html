写在前面

昨天跟着教程写了一下gulp对开发环境自动化的构建，发现比webpack（当前最火的前端资源模块化管理和打包工具）入门简单多了，所以先写gulp再写webpack

  • 当前使用gulp版本为3.9.1

gulp的Api

看英文文档主要使用的好像一共就5个api

  • gulp.src //读取文件
  • gulp.dest //输出文件
  • gulp.task //定义任务
  • gulp.watch //监听文件改变
  • gulp.run(tasks…) //尽可能多的并行运行多个task

gulp之代码压缩

上边讲的只是单纯讲一个文件按照原样的转移到另外一个文件中，现在我们试着对html js css等文件进行压缩

gulp-htmlmin压缩html

首先在开发环境安装一个模块 gulp-htmlmin
在命令行中输入 npm install gulp-htmlmin –save-dev
